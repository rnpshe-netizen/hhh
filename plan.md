# 프로젝트 설계도 (Plan)

## 🚨 그라운드 룰 (Ground Rule)
> **새로운 대화 세션이 시작될 때마다, 반드시 아래 순서를 따른다.**
> 1. `plan.md`를 처음부터 끝까지 읽고 프로젝트의 목적, 구조, 기술 스택, 매니저님 결정 사항을 파악한다.
> 2. `progress.md`를 읽고 현재 진행 단계와 완료/미완료 항목을 확인한다.
> 3. 위 두 문서를 모두 숙지한 후에만 다음 작업 단계를 진행한다.
> 4. 작업 중 발생하는 모든 변경 사항과 매니저님의 결정은 즉시 `plan.md` 또는 `progress.md`에 기록한다.
> 5. **[절대 준수]** 실제 코드를 짜지 않고 스크립트로 데이터만 조회/검증/단순 확인한 작업조차도, 채팅창에 보고하기 전에 무조건 두 문서(plan/progress)에 먼저 진행 내역을 기록한 뒤 보고한다.

## 1. 프로젝트 목적
**국제코칭훈련원 회원·과정 통합 관리 시스템** 구축
- 현재 과정별로 흩어져 있는 엑셀 수료 현황을 **하나의 웹 대시보드**에서 통합 관리
- 회원 정보와 수료 이력을 한눈에 파악하고, 축하/안내 메시지를 발송하며, 추후 자격증 자동 발급까지 확장

## 2. 요구사항 분석

### 2-1. 현재 상태 (AS-IS)
| 구분 | 현재 방식 |
|------|-----------|
| 수료 현황 관리 | 과정별 개별 엑셀 관리 (7개 파일) |
| 수료증 발급 | 수동 발급 |
| 회원 소통 | 개별 수동 연락 |

### 2-2. 관리 중인 과정 목록 (엑셀 분석 결과)
| 파일 | 과정명 | 데이터 건수 |
|------|--------|------------|
| 1 | TRAIN 기초 | 1,500건 |
| 2 | TRAIN 심화 | 304건 |
| 3 | TRAIN Advanced | 337건 |
| 4 | TRAIN 기초+심화 | 751건 |
| 5 | 가족코칭 전문가 | 502건 |
| 6 | 가족코칭지도사 2급 | 2,370건 |
| 7 | 라이프코칭 | 221건 |

> ⚠️ 6번 파일에 주민번호가 포함되어 있어, DB 이관 시 **주민번호는 제외** 처리합니다.
> ⚠️ 동명이인 구분이 어려워, 우선 **이름 기준**으로 회원을 묶고 추후 연락처/이메일로 정확도를 높입니다.

### 2-3. 목표 상태 (TO-BE)
- **Phase 1** (MVP): 회원 통합 관리 + 과정별 수료 현황 대시보드
- **Phase 2**: 축하 문자 / 과정 추천 안내 (문자, 이메일, 카카오톡)
- **Phase 3**: 수료증·자격증 자동 발급

## 3. 기술 스택
| 영역 | 기술 | 비유 |
|------|------|------|
| 프론트엔드 (화면) | Next.js + React | 사용자가 보는 웹 화면 |
| 백엔드 (서버/DB) | Supabase (PostgreSQL) | 데이터를 안전하게 저장하는 클라우드 창고 |
| 배포 (웹 주소) | Vercel | 만든 웹을 인터넷에 공개하는 서비스 |
| 스타일링 | Vanilla CSS | 화면 디자인 |

## 4. Phase 1 (MVP) 세부 계획
> ⚠️ 우선 Phase 1에만 집중합니다. Phase 2, 3은 Phase 1 완성 후 확장합니다.

### 4-1. 핵심 기능
1. **회원 관리**: 이름, 연락처, 이메일 등 회원 정보 등록/조회/수정/삭제
2. **과정 관리**: 운영 중인 코칭 과정 목록 관리
3. **수료 현황 대시보드**: 회원별 수료 과정 한눈에 보기 (표 + 뱃지 형태)
4. **엑셀 데이터 임포트**: 기존 7개 엑셀 파일의 데이터를 Supabase DB로 일괄 이관

### 4-2. 데이터베이스 설계
- **`members`** (회원): id, name, phone, email, created_at
- **`courses`** (과정): id, name, category(수료증/자격증), description
- **`completions`** (수료 기록): id, member_id, course_id, registration_no, issued_date, cohort, note

### 4-3. 웹 페이지 구성
1. **대시보드** (`/`) — 회원별 수료 현황을 뱃지/표로 한눈에 보기
2. **회원 목록** (`/members`) — 회원 검색, 등록, 수정, 삭제
3. **과정 관리** (`/courses`) — 과정 목록 관리

### 4-4. 작업 순서
1. ✅ Next.js 프로젝트 초기 세팅 (완료)
2. ✅ Supabase 프로젝트 연동 및 DB 테이블 생성 (완료)
3. ✅ 엑셀 데이터 이관 스크립트 작성 및 실행 (완료)
4. ✅ 회원 관리 / 과정 관리 / 대시보드 화면 개발 (완료)
5. ✅ Vercel 배포 (완료)

## 5. Phase 1.5 (관리자 기능 고도화) 세부 계획

### 5-1. 핵심 기능 5가지
1. **회원 정보 수정/삭제**: 이름, 연락처 수정 및 영구 삭제 (연쇄 삭제 경고 포함)
2. **과정 수동 발급 관리**: 회원 상세창에서 수료 과정 즉시 연결 및 발급 취소 기능
3. **과정 비활성화(숨김)**: 과거 과정 삭제 방지 및 숨김 스위치 (DB `is_active` 컬럼 추가 필요)
4. **엑셀 추출 (CSV)**: 현재 검색된 회원 명단을 엑셀 파일로 일괄 다운로드
5. **데이터 중복 방지**: 회원 추가/수정 시 동일 연락처 중복 체크 경고
