# ICTI-MIS 평가 기준표

모든 에이전트(Generator)와 테스터(Evaluator)가 공유하는 품질 기준.

---

## 채점 항목

### 1. 기능 정확성 (비중: 40%)
요구된 기능이 정확하게 동작하는가?

합격: CRUD 정상 작동, 필터/검색 결과 정확, DB 반영 확인, 에러 핸들링 존재
불합격: 버튼 클릭 무반응, 데이터 불일치, 콘솔 에러 발생, 페이지 깨짐

### 2. UI/UX 일관성 (비중: 25%)
기존 시스템과 통일된 디자인인가?

합격: 기존 카드/테이블/버튼 스타일과 동일, 색상 팔레트 준수 (primary=#4A90E2, success=#27AE60), 폰트/간격 일관
불합격: 기존 페이지와 이질적, 스타일 하드코딩 불일치, 레이아웃 깨짐

### 3. 데이터 무결성 (비중: 20%)
DB 작업이 안전한가?

합격: 삭제 시 경고 표시, 활동 로그 기록됨, CASCADE 동작 정확, 입력 검증 존재
불합격: 무경고 삭제, 로그 누락, 데이터 오염 가능성, 입력 검증 없음

### 4. 성능 (비중: 15%)
페이지 로딩과 작업 반응이 적절한가?

합격: 대시보드 3초 이내 로딩, 필터 변경 즉시 반영, 불필요한 재렌더링 없음
불합격: 5초 이상 로딩, 필터 변경 후 빈 화면, 무한 로딩

---

## 판정

가중 점수 = (기능×0.4) + (UI/UX×0.25) + (데이터×0.2) + (성능×0.15)

- **7.0 이상** → PASS (합격)
- **5.0~6.9** → CONDITIONAL (조건부 — 피드백 반영 후 재검수)
- **5.0 미만** → FAIL (불합격)
- **기능 정확성 4점 이하** → 무조건 FAIL

```
**전체 판정**: [PASS / CONDITIONAL / FAIL]
**가중 점수**: X.X / 10.0

**항목별 점수**:
- 기능 정확성: X/10 — [한 줄 코멘트]
- UI/UX 일관성: X/10 — [한 줄 코멘트]
- 데이터 무결성: X/10 — [한 줄 코멘트]
- 성능: X/10 — [한 줄 코멘트]

**구체적 개선 지시**:
1. [어디를 어떻게 고칠 것 — 파일명:라인 포함]
2. [어디를 어떻게 고칠 것]

**방향 판단**: [현재 방향 유지] 또는 [근본적 재작업 필요]
```
